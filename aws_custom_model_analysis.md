# AWS カスタムモデル vs 現在のAPI利用 詳細分析

## 現在の状況
- **月額費用**: 15,000円（Claude API）
- **実行回数**: 150回/月（1日5回）
- **処理内容**: 画像2枚 + PDF 80,000文字

## AWS学習可能モデルの分析

### 1. Amazon SageMaker（フルカスタムモデル）

#### 初期投資
```
モデル訓練費用:
- ml.p3.2xlarge (GPU): 580円/時間
- 訓練時間: 50-100時間
- 訓練コスト: 29,000円～58,000円
- データ準備: 10,000枚以上のチャート画像必要
```

#### 月間運用費
```
推論エンドポイント:
- ml.t2.medium: 8円/時間
- 月間: 8円 × 24時間 × 30日 = 5,760円
- ストレージ: 500円
合計: 約6,300円/月
```

#### 損益分岐点
- 初期投資回収: 4-8ヶ月
- **年間削減額**: 約107,000円

### 2. Amazon Bedrock（ファインチューニング）

#### コスト構造
```
ファインチューニング:
- 基本料金: $40-400（6,000円～60,000円）
- データ量により変動

推論コスト:
- $0.012/1000トークン（入力）
- $0.016/1000トークン（出力）
- 月間: 約27,000円（現在より高額）
```

#### 評価
❌ **推奨しない** - 現在のAPIより高額

### 3. Amazon Rekognition Custom Labels

#### 特徴
- チャートパターン認識に特化可能
- 最小250枚の画像で訓練開始

#### コスト
```
訓練: $20/時間 × 10-20時間 = 7,600円～15,000円
推論: $4/時間 = 月間30,000円（常時起動の場合）
```

#### 評価
❌ 画像のみで、テキスト分析は別途必要

### 4. オープンソース + EC2構築

#### 構成案
```
モデル: Llama 3.2 Vision (11B)
インスタンス: g5.2xlarge (GPU付き)
月額: 約50,000円（オンデマンド）
     約15,000円（リザーブドインスタンス）
```

#### メリット
- 完全にカスタマイズ可能
- 制限なし
- データプライバシー確保

## 推奨プラン

### Phase 1: 即効性重視（0-3ヶ月）
```python
# hybrid_solution.py実装
# Gemini無料枠 + Claude Haiku
月額: 500円（97%削減）
```

### Phase 2: データ蓄積（3-12ヶ月）
```
- 現在の分析結果を蓄積
- チャート画像とラベルを保存
- 10,000データポイント目標
```

### Phase 3: カスタムモデル移行（12ヶ月後）
```
Amazon SageMaker導入:
- 初期投資: 50,000円
- 月額: 6,300円（58%削減）
- 年間削減: 107,000円
```

## 結論

### 短期的には
**Gemini無料枠 + Claude Haikuのハイブリッド**が最適
- 月額500円で即実装可能
- 品質維持

### 長期的には
**Amazon SageMakerへの移行**を推奨
- 1年後に十分なデータが蓄積
- 初期投資は5ヶ月で回収
- その後は年間10万円以上の削減

### AWSカスタムモデルは高額ではない
むしろ長期的には**大幅なコスト削減**が可能です。
ただし、初期データの蓄積が成功の鍵となります。
