# 検証日数表示機能の実装完了

## 実装内容

### 1. 検証日数トラッカー (`src/verification_tracker.py`)
- Phase1開始日を自動記録
- 検証○日目を計算・表示
- 進捗率の計算（Phase1: 90日）
- フェーズ情報の管理

### 2. 各プラットフォームでの表示

#### WordPress/ブログ
- タイトル: `【USD/JPY】2024年3月15日 朝の相場分析 - 検証15日目`
- 本文下部: `<strong>検証15日目</strong>`として表示

#### X (Twitter)
- ハッシュタグに追加: `#検証15日目`
- 例: `#USDJPY #ドル円 #FX学習 #検証15日目`

#### Notion
- 分析日時欄: `分析日時: 2024年3月15日 14:30 - 検証15日目`

#### Slack
- タイトル: `*FX分析完了 (Volmanメソッド) - 検証15日目*`

### 3. 使用方法

#### 初回実行時
初回実行時に自動的に `phase1_start_date.json` が作成され、その日が検証1日目となります。

```json
{
  "start_date": "2024-03-15T00:00:00",
  "phase": "Phase1",
  "description": "FXスキャルピング検証プロジェクト"
}
```

#### 日次実行
毎日の実行時に自動的に検証日数がカウントアップされ、すべての投稿に反映されます。

### 4. テスト方法

```bash
# ローカルテスト
python test_verification_day.py

# 実際の投稿テスト（検証日数付き）
python execute_complete_posting.py
```

### 5. AWS環境での動作

AWS ECS/Lambda環境でも同様に動作します。`phase1_start_date.json`ファイルは以下の場所に保存されます：
- ローカル: プロジェクトルート
- AWS: EFSまたはS3（永続化が必要な場合）

### 6. フェーズ移行

- **Phase1 (1-90日)**: データ収集・検証期間
- **Phase2 (91-180日)**: ブログ収益化期間  
- **Phase3 (181日以降)**: AIシグナルサービス期間

各フェーズは自動的に判定され、表示に反映されます。

## 注意事項

1. `phase1_start_date.json`を削除すると、検証日数がリセットされます
2. 手動で日付を変更する場合は、JSONファイルを直接編集してください
3. AWS環境では、ファイルの永続化を考慮してください（S3やEFSの使用）
