## 📝 ドキュメント管理ルール

1. **命名規則**
   - `XX_document_name.md` 形式（XXは2桁の連番）
   - 新規追加時は最後の番号+1を使用
   - ファイル名は内容を表す分かりやすい名前に

2. **カテゴリ分類**
   - セットアップ・導入関連: 01-05
   - コスト・AI関連: 06-10
   - デプロイ・運用関連: 11-14
   - 品質・テスト関連: 15-17
   - 実装計画: 18-

3. **更新時の注意**
   - このインデックスファイルも同時に更新
   - 削除時は番号を詰めずに欠番とする
   - 大規模な再編成時のみ番号を振り直す

4. **ドキュメントの追加方法**
   ```bash
   # 新しいドキュメントを追加する場合
   cd docs/
   # 最新の番号を確認して、次の番号を使用
   touch 19_new_document_name.md
   # このインデックスファイルを更新
   ```
